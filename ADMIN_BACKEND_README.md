# 管理员后台系统使用说明

## 概述

这是一个完整的签证申请系统管理员后台，提供了用户管理、订单管理、退款处理、系统监控等功能。

## 功能模块

### 1. 仪表板 (Dashboard)
- **系统概览**: 显示关键指标和统计数据
- **实时监控**: 系统状态和健康检查
- **待处理事项**: 退款申请、文档审核等
- **最近订单**: 最新处理的订单列表

### 2. 用户管理
- **用户列表**: 查看所有注册用户
- **用户状态**: 活跃、非活跃、禁用状态管理
- **角色管理**: 普通用户和管理员角色分配
- **搜索过滤**: 按用户名、邮箱、状态等条件筛选
- **用户操作**: 查看详情、编辑、启用/禁用、删除

### 3. 订单管理
- **订单列表**: 所有签证申请订单
- **订单状态**: 待处理、处理中、已批准、已拒绝、已完成
- **订单详情**: 查看订单完整信息
- **订单操作**: 状态更新、备注添加

### 4. 退款管理
- **退款申请**: 查看用户退款申请
- **退款审核**: 批准或拒绝退款请求
- **处理意见**: 添加管理员处理意见
- **退款统计**: 退款金额和状态统计

### 5. 系统监控
- **服务状态**: 数据库、API、监控服务等状态
- **系统日志**: 查看系统运行日志
- **性能监控**: 系统性能指标

### 6. 数据统计
- **用户统计**: 用户增长、活跃度分析
- **订单统计**: 订单量、收入统计
- **趋势分析**: 数据趋势图表

### 7. 系统设置
- **系统配置**: 修改系统参数
- **邮件设置**: 邮件服务配置
- **支付设置**: 支付网关配置

### 8. 安全日志
- **登录日志**: 用户登录记录
- **操作日志**: 管理员操作记录
- **安全事件**: 异常登录和操作

## 技术架构

### 前端技术栈
- **Next.js 14**: React框架
- **TypeScript**: 类型安全
- **Tailwind CSS**: 样式框架
- **shadcn/ui**: UI组件库
- **Lucide React**: 图标库

### 后端技术栈
- **Next.js API Routes**: 后端API
- **Prisma**: 数据库ORM
- **PostgreSQL**: 主数据库
- **NextAuth.js**: 身份认证

### 数据库设计
- **用户表**: 用户基本信息、角色、状态
- **订单表**: 签证申请订单信息
- **退款表**: 退款申请和处理记录
- **系统日志表**: 操作和系统日志

## 安装和部署

### 1. 环境要求
- Node.js 18+
- PostgreSQL 12+
- npm 或 yarn

### 2. 安装依赖
```bash
npm install
```

### 3. 环境配置
创建 `.env.local` 文件:
```env
DATABASE_URL="postgresql://username:password@localhost:5432/visa_assistant"
NEXTAUTH_SECRET="your-secret-key"
NEXTAUTH_URL="http://localhost:3000"
```

### 4. 数据库迁移
```bash
npx prisma migrate dev
npx prisma generate
```

### 5. 启动开发服务器
```bash
npm run dev
```

## 使用流程

### 1. 管理员登录
1. 访问 `/admin/login`
2. 输入管理员邮箱和密码
3. 点击登录按钮

### 2. 用户管理
1. 在侧边栏点击"用户管理"
2. 查看用户列表和统计信息
3. 使用搜索和过滤功能
4. 点击操作按钮进行用户管理

### 3. 退款处理
1. 在侧边栏点击"退款管理"
2. 查看待处理的退款申请
3. 点击"查看详情"查看完整信息
4. 添加处理意见并批准或拒绝

### 4. 系统监控
1. 在侧边栏点击"系统监控"
2. 查看各项服务状态
3. 检查系统日志
4. 监控性能指标

## 常见问题

### Q: 无法登录管理员后台？
A: 检查以下几点：
1. 确认使用正确的管理员账号
2. 检查数据库连接是否正常
3. 确认环境变量配置正确

### Q: 用户数据不显示？
A: 检查以下几点：
1. 确认数据库中有用户数据
2. 检查API接口是否正常
3. 查看浏览器控制台错误信息

### Q: 退款处理失败？
A: 检查以下几点：
1. 确认退款申请状态
2. 检查处理意见是否填写
3. 确认API接口权限

## 开发说明

1. 修改 `tailwind.config.js` 配置
2. 在组件中使用Tailwind CSS类
3. 创建自定义CSS文件（如需要）

### 3. 数据库操作
1. 使用Prisma Client进行数据库操作
2. 创建数据库迁移文件
3. 更新Prisma schema
