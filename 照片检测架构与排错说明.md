## 三、常见失败原因与处理

### 1. Playwright 浏览器未安装（最常见）

**错误**：`Executable doesn't exist at ... firefox.exe` 或 `chromium`

**处理**：
```powershell
# Python 环境安装 Chromium（主用 services/photo-checker）
python -m playwright install chromium
```

### 2. 外部 API 仍用 Firefox（usvisa/checker.py）

`app/api/usa-visa/usvisa/checker.py` 中有 `random.choice([chromium, firefox])`，若本地失败会走外部 API，外部若选到 Firefox 也会报错。已建议改为仅用 Chromium。

### 3. CEAC 官网访问异常

- **国内网络**：`ceac.state.gov` 可能被限速或拦截，需代理/VPN
- **页面结构变化**：CEAC 若改版，选择器可能失效

### 4. Python 依赖缺失

```powershell
pip install pillow playwright
python -m playwright install chromium
```

### 5. 路径与执行环境

- Windows 路径含空格时需加引号（已在 route 中处理）
- 确认系统 PATH 中 `python` 可正常执行

## 四、DS-160 Security and Background Part 1 卡死问题

该页有三个 Yes/No 问题（传染病、精神障碍、毒品），点 Next 后常卡死或无法跳转。

### 原因
- **第一题未正确选中**：CEAC 使用 ASP.NET CustomValidator，第一题（传染病）必须明确选中并触发 `setDirty` / `ShowHideDiv`，否则验证失败，Next 不生效或卡死。
- **网络/会话**：ceac.state.gov 国内可能慢或超时；会话超时约 20 分钟无操作会失效。

### 人工填表时
1. **先选第一题**：务必先点「传染病」的 No，等 1–2 秒，再选后两题。
2. **检查红色错误**：若题目旁出现红色错误图标，说明未通过验证，重新点选该题的 No。
3. **先点 Save 再 Next**：部分情况下先点 Save 保存，再点 Next，可减少卡死。
4. **换浏览器/网络**：用 Chrome 无痕模式、关闭广告拦截；若在国内，尝试稳定 VPN。

### 自动填表（ds160_server.py）
已优化：选中时触发 `setDirty` 和 `ShowHideDiv`，并在点 Next 前校验三题均选中；若未选中则重试勾选。
