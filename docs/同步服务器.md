## 一键部署

在项目根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\deploy-volcengine.ps1
```

这个脚本会做这些事：

1. 打包当前项目代码
2. 上传到 `101.96.202.43`
3. 解压到 `/opt/visa-assistant`
4. 用 `.env.server` 执行 `npm run build`
5. 重启线上服务
6. 做一次本机 `127.0.0.1:3000` 的健康检查

## 常用参数

默认直接用当前服务器。如果以后换机器，可以传参：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\deploy-volcengine.ps1 `
  -ServerIp 101.96.202.43 `
  -SshUser root `
  -KeyPath "d:\360Downloads\download_chrome\vistoria.pem"
```

只上传代码，不构建重启：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\deploy-volcengine.ps1 -SkipBuild
```

## 手工部署命令

如果脚本失效，按这个顺序手工执行。

### 1. 打包本地代码

```powershell
tar `
  --exclude=.git `
  --exclude=node_modules `
  --exclude=.next `
  --exclude=temp `
  --exclude=storage `
  --exclude=.test-dist `
  --exclude=.env.local `
  --exclude=.env.server `
  -czf $env:TEMP\visa-assistant-deploy.tar.gz .
```

### 2. 上传到服务器

```powershell
scp -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null `
  -i "d:\360Downloads\download_chrome\vistoria.pem" `
  $env:TEMP\visa-assistant-deploy.tar.gz `
  root@101.96.202.43:/tmp/visa-assistant-deploy.tar.gz
```

### 3. 服务器解压

```powershell
ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null `
  -i "d:\360Downloads\download_chrome\vistoria.pem" `
  root@101.96.202.43 `
  "mkdir -p /opt/visa-assistant && tar -xzf /tmp/visa-assistant-deploy.tar.gz -C /opt/visa-assistant && rm -f /tmp/visa-assistant-deploy.tar.gz"
```

### 4. 构建并重启

```powershell
ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null `
  -i "d:\360Downloads\download_chrome\vistoria.pem" `
  root@101.96.202.43 `
  "bash -lc 'cd /opt/visa-assistant && set -a && source .env.server && set +a && npm run build' && systemctl restart visa-web.service visa-trip-generator.service visa-explanation-letter.service visa-material-review.service visa-tls-monitor.service"
```

### 5. 验证

```powershell
ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null `
  -i "d:\360Downloads\download_chrome\vistoria.pem" `
  root@101.96.202.43 `
  "systemctl is-active visa-web.service visa-trip-generator.service visa-explanation-letter.service visa-material-review.service visa-tls-monitor.service && curl -I -s http://127.0.0.1:3000/ | head -n 1"
```

## 法签相关环境说明

法签自动化相关 Python 任务现在统一要求：

- 服务器上存在 `python`
- `python` 能指向 `python3`
- 已安装 Python 依赖
- 已安装 Playwright Chromium

当前服务器如果要补环境，用这些命令：

```bash
ln -sf /usr/bin/python3 /usr/local/bin/python
python -m pip install --break-system-packages --ignore-installed pandas openpyxl playwright selenium requests
python -m playwright install chromium
```

## 常见排查

### 1. 报 `浏览器初始化失败`

先查：

```bash
python -m playwright install chromium
```

### 2. 报 `No module named ...`

先补 Python 依赖：

```bash
python -m pip install --break-system-packages --ignore-installed pandas openpyxl playwright selenium requests
```

### 3. 构建后页面没更新

先确认：

```bash
systemctl restart visa-web.service
systemctl is-active visa-web.service
```

### 4. 法签任务只显示退出码

去看对应任务目录里的：

- `runner_stdout.log`
- `runner_stderr.log`

目录位置通常在：

- `/opt/visa-assistant/temp/french-visa-fill-receipt/...`
- `/opt/visa-assistant/temp/french-visa-submit-final/...`
- `/opt/visa-assistant/temp/french-visa-register/...`
