# 美签 Excel 模板对比分析

## 一、文件概览

| 项目 | 美签办理基本信息表（正常） | 2026-美签办理基本信息表（易出错） |
|------|---------------------------|-----------------------------------|
| Sheet1 行数 | 144 行 | 159 行 |
| Sheet1 列数 | **4 列** | **19 列** |
| 结构 | 基本信息 / Field / 填写内容 | 同左 + 多列 Unnamed 空列 |

---

## 二、结构差异

### 1. 列结构
- **美签**：4 列，结构紧凑
- **2026**：19 列，存在大量空列（Unnamed: 4 ~ Unnamed: 18），可能是合并单元格或模板占位导致

### 2. 行顺序
- **2026** 在「是否有曾用名」后多了两行：**姓(拼音）**、**名(拼音）**
- 后续所有行相对美签整体下移 2 行
- 代码按「基本信息」列匹配，顺序变化不影响读取

---

## 三、字段与值差异（可能导致问题）

### 1. Yes/No 类字段

| 基本信息 | 美签 | 2026 | 影响 |
|----------|------|------|------|
| 是否有曾用名 | `False`（布尔） | `YES`（字符串） | 2026 会点 Yes 并填写曾用名，但**缺少 曾用名姓氏/曾用名名字**，会填空字符串 |
| 是否去过美国 | No | **YES** | 2026 会进入「去过美国」逻辑，但字段名不一致 |
| 是否曾有美国签证 | No | **YES** | 会填写签证信息 |
| 是否有美国驾照 | No | **YES** | 会填写驾照信息 |
| 是否被拒签 | No | **YES** | 会填写拒签说明 |
| 是否有工作经历 | No | NO | 基本等效 |

### 2. 字段名不一致（2026 独有或不同）

| 2026 字段名 | 美签对应字段 | 说明 |
|-------------|--------------|------|
| 姓(拼音）| 曾用名姓氏 | **已支持**：程序会读取并填入曾用名姓氏 |
| 名(拼音）| 曾用名名字 | **已支持**：程序会读取并填入曾用名名字 |
| 到达美国的日期 | 去美年份 + 去美月份 + 去美日期 | 2026 用单一日期，需拆分 |
| 在美国呆的单位 | 无 | DAY(S) |
| 在美国呆的数量 | 无 | 7 |
| 去美城市 | （美签有） | 2026 无此行 |
| 去美年月日 | （美签有） | 2026 无此行 |
| 去美所在州 | （美签有） | 2026 无此行 |

### 3. 其他差异
- **个人邮箱**：美签 `147434834444@qq.com`，2026 `ukvisa20242024@163.com`
- **去过的国家**：2026 填了 7 个国家（含 去过的国家6=阿尔及利亚、去过的国家7=阿尔巴尼亚），美签为 5 个
- **国家数量**：美签 5，2026 7

## 四、与 Application Error 的可能关联

1. **曾有明=Yes 但曾用名姓氏/名字为空**（已修复）
   - 2026 表有此情况，CEAC 在 Travel Companions 点击 Next 时会做全量校验并报 Application Error
   - **程序已自动处理**：若 曾有明=Yes 但姓氏/名字均为空，自动改为 No 并记录日志

2. **字段缺失或命名不匹配**
   - 2026 缺少 去美年份/去美月份/去美日期 等，仅有 到达美国的日期
   - 程序可能拿不到正确的 previous_us_travel_* 数据，表单状态不一致

3. **CEAC 服务器稳定性**
   - 更复杂的提交（更多国家、更多 Yes 分支）可能增大请求体
   - 服务器异常或超时也可能表现为 Application Error

---

## 五、建议

### 1. 统一 2026 模板字段名
在 2026 表中补全并与美签对齐：

- 是否有曾用名 = YES 时，必须有：**曾用名姓氏**、**曾用名名字**
- 是否去过美国 = YES 时，使用：**去美年份**、**去美月份**、**去美日期**、**去美国家**、**去美城市**、**去美州**，而不是 到达美国的日期 / 在美国呆的单位 / 在美国呆的数量

### 2. 精简 2026 表结构
- 删除多余空列，保持 4 列结构（基本信息 / Field / 填写内容）
- 避免合并单元格导致的列数异常

### 3. 程序兼容
在 `process_excel_data` 中为 2026 特有字段做映射：

- `到达美国的日期` → 拆分为 previous_us_travel_year / month / day
- `在美国呆的数量` → 用于 previous_us_travel 相关逻辑

## 七、Previous U.S. Travel 与签证相关字段（本次报错修复）

### 1. 报错原因（error_html_1774252929）

- **Year has not been completed**：签证遗失=Yes 时，必须填写「签证遗失或被盗年份」
- **Has your U.S. Visa ever been lost or stolen? is invalid**：Explain 文本框仅允许 ASCII 字符（A-Z, 0-9, #, $, *, %, & 等），**禁止中文**
- **Has your U.S. Visa ever been cancelled or revoked? is invalid**：同上，Explain 禁止中文

### 2. Excel 需补充/修正的字段

| 基本信息（Excel 列名） | 填写内容格式 | 说明 |
|------------------------|--------------|------|
| 签证是否遗失或被盗 | Yes / No | 若填 Yes，**必须**填写下面两项 |
| 签证遗失或被盗年份 | 2020 | 4 位数字，**必填**当遗失=Yes。若为空，程序自动改为 No |
| 签证遗失或被盗说明 | Visa was lost in 2020 | **仅英文**，禁止中文。CEAC 允许：A-Z, 0-9, #, $, *, %, &, 空格等 |
| 签证是否被取消或吊销 | Yes / No | |
| 签证被取消或吊销说明 | N/A | **仅英文**，禁止中文。同上 |

### 3. 去过美国相关（支持 2026 单一日期格式）

| 基本信息 | 填写示例 | 说明 |
|----------|----------|------|
| 到达美国的日期 | 2020-09-21 或 2020/9/21 | 程序会拆分为 年/月/日 填入 |
| 在美国呆的数量 | 7 | 数字 |
| 在美国呆的单位 | D 或 Day(s) | D=天, W=周, M=月, Y=年 |

### 4. 被拒签、移民申请说明

- **被拒签原因**、**移民签证申请说明**：同样**仅英文**，禁止中文，程序会自动过滤非法字符。

### 5. 前单位 / 工作经历（Previous Employer）

| 基本信息（Excel 列名） | 填写示例 | 说明 |
|------------------------|----------|------|
| 前单位职位 | Software Engineer | CEAC: Job Title，职位/职务 |
| 前学校/单位 | ABC Company | 前雇主名称 |
| 前单位工作内容 | Developed web applications | 工作职责描述 |
