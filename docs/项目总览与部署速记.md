## 1. 项目是什么

这是一个签证业务工作台，核心不是单一页面，而是一整套签证业务系统。

当前主要覆盖两条业务线：
- 美国签证
- 法国申根签证

系统已经做过的主要能力：
- 申请人档案管理
- 美签 Excel 审核
- DS-160 自动化
- 美签面试必看生成与审核
- 法签自动化
- 法签综合材料审核
- 行程单生成
- 解释信生成
- 材料审核
- 企业微信微盘接入预留

## 7. 当前重要业务约定

### 法签综合材料审核
- `申根 Excel` 是标准信息源
- 其他材料先对 `申根 Excel`
- 材料之间也会互相审核
- 只要有核心不一致，结论就是 `不可递签`
- `机票` 和 `保险` 缺失不会直接拦截，但会提示缺少材料

当前重点对比：
- FV申请回执单
- TLS预约单
- 行程单
- 酒店订单
- 机票
- 保险

已经特别处理的规则：
- `申根国家` 不参与综合审核结论
- `FV申请回执单` 对 `申根 Excel` 要求更严格，关键字段不能错
- Booking 酒店单解析要优先读头部 `CHECK-IN / CHECK-OUT / NIGHTS`
- 酒店地址比对不能只用整串包含，要按核心地址 token 比
- 如果上传的是 `TLS 注册表` 而不是标准申根表，不能误当标准源

### 美签面试必看
- 标准源：美签 Excel
- 对照源：DS-160 结构化结果、面试必看结构化结果
- 结论只有：
  - `审核通过`
  - `需要修改`

已经修过的重要问题：
- Excel 日期序列号如 `46152`、`46157` 必须先转成真实日期
- 面试必看模板文件必须在：
  - `storage/templates/us-visa/interview-brief-template.docx`
- 服务器需要安装：
  - `python-docx`

### 行程单生成
- 当前已经改回 `纯模板驱动`
- 真正生效的模板是：
  - `app/trip_generator/template.docx`
- 生成流程是：
  - 模板生成 `docx`
  - 再转成 `pdf`
- 线上服务器必须有 `LibreOffice`

### 解释信生成
- 新生成的 Word/PDF 要优先归档到申请人档案
- 下载与预览应优先使用档案归档链接
- 文件名不能再是默认的 `pdf_en.pdf / word_en.docx`

## 9. 常见问题排查

### 1. 法签任务只显示退出码
去对应任务目录看：
- `runner_stdout.log`
- `runner_stderr.log`

常见目录：
- `/opt/visa-assistant/temp/french-visa-fill-receipt/...`
- `/opt/visa-assistant/temp/french-visa-submit-final/...`
- `/opt/visa-assistant/temp/french-visa-register/...`

### 2. 浏览器初始化失败
优先检查：

```bash
python -m playwright install chromium
```

### 3. Python 模块缺失
优先补依赖：

```bash
python -m pip install --break-system-packages --ignore-installed pandas openpyxl playwright selenium requests PyMuPDF pypdf python-docx
```

### 4. 行程单 PDF 生成失败
先确认服务器存在：
- `libreoffice`
- `soffice`

### 5. 面试必看模板缺失
确认文件存在：
- `/opt/visa-assistant/storage/templates/us-visa/interview-brief-template.docx`
