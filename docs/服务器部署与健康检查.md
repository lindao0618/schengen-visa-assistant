# 服务器部署与健康检查

本文档用于线上部署、域名绑定后排障，以及确认服务是否正常运行。

## 1. 推荐部署顺序

1. 拉取最新代码。
2. 安装依赖：`npm ci`
3. 确认 `.env.production` 或服务器环境变量已配置。
4. 构建：`npm run build`
5. 启动：`npm run start`
6. 检查健康接口：`curl -i http://127.0.0.1:3000/api/health`
7. 再通过域名检查：`curl -i https://www.vistoria618.top/api/health`

## 2. 必填环境变量

健康检查会检查以下关键变量是否存在，但不会输出密钥内容：

- `DATABASE_URL`
- `NEXTAUTH_SECRET`
- `NEXTAUTH_URL`

如果 `/api/health` 返回 `503` 且 `status` 为 `degraded`，优先检查这三个变量。

生产环境建议同时确认：

- `NEXTAUTH_URL=https://www.vistoria618.top`
- `TRIP_GENERATOR_URL=http://127.0.0.1:8002`
- `EXPLANATION_LETTER_URL=http://127.0.0.1:8003`
- `MATERIAL_REVIEW_URL=http://127.0.0.1:8004`
- `TLS_MONITOR_URL=http://127.0.0.1:8005`
- 邮件、验证码、OCR、企微等第三方密钥按 `.env.example` 配齐。

## 3. 健康检查接口

接口地址：

```bash
GET /api/health
```

正常返回示例：

```json
{
  "status": "ok",
  "service": "visa-assistant",
  "timestamp": "2026-04-26T00:00:00.000Z",
  "uptimeSeconds": 123,
  "environment": "production",
  "checks": {
    "databaseUrl": true,
    "nextAuthSecret": true,
    "nextAuthUrl": true
  }
}
```

说明：

- `200` 表示关键运行变量齐全。
- `503` 表示服务进程还活着，但关键配置缺失。
- 接口不检查数据库连通性，避免健康检查本身拖慢或压垮数据库；数据库连接问题应通过应用日志继续排查。

## 4. Nginx 反向代理检查

确认 Nginx 指向 Next.js 端口，例如：

```nginx
server {
  listen 80;
  server_name vistoria618.top www.vistoria618.top;

  location / {
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
}
```

检查命令：

```bash
sudo nginx -t
sudo systemctl reload nginx
curl -i http://127.0.0.1:3000/api/health
curl -i http://vistoria618.top/api/health
```

## 5. 常见打不开排查顺序

1. 服务器进程是否存在：`ps aux | grep next`
2. 本机端口是否监听：`ss -lntp | grep 3000`
3. 本机健康检查是否正常：`curl -i http://127.0.0.1:3000/api/health`
4. Nginx 配置是否正确：`sudo nginx -t`
5. 域名 DNS 是否解析到服务器公网 IP。
6. 安全组/防火墙是否放行 `80` 和 `443`。
7. HTTPS 证书是否正常：`sudo certbot certificates`
8. 应用日志是否有环境变量、数据库、权限错误。

## 6. 回滚建议

每次上线前先记录当前提交：

```bash
git rev-parse --short HEAD
```

如果新版本异常：

```bash
git log --oneline -10
git checkout <上一个正常提交>
npm ci
npm run build
npm run start
```

如果使用 `systemd` 管理服务，重启命令通常是：

```bash
sudo systemctl restart visa-next
sudo journalctl -u visa-next -n 100 --no-pager
```
