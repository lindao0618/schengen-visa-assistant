# OpenClaw 员工操作手册

若需通过 HTTP/API 或 Agent 调用本站接口（Method、路径、slot 列表等），见 **[Agent HTTP API 参考](./AGENT_HTTP_API_REFERENCE.md)**。

## 1. 项目概览

这个项目目前主要有两条业务线：

- 美签自动化
- 申根自动化，目前重点是法国

常用页面入口：

- 申请人档案中心：`/applicants`
- 美签操作页：`/usa-visa`
- 法签自动化页：`/schengen-visa/france/automation`

当前系统规则：

- 所有自动化都在服务器端运行
- 美签自动化默认无头运行
- 法签自动化默认无头运行
- 员工不需要也不应该手动操作服务器浏览器窗口

## 2. 总操作原则

所有业务都建议先建申请人档案，再做自动化。

推荐顺序：

1. 创建或选择申请人档案
2. 把资料上传到档案中心
3. 进入美签或法签页面执行自动化
4. 在任务列表里查看结果
5. 回申请人档案中心确认最终材料是否已归档

## 3. 申请人档案中心

页面：

- `/applicants`

这个页面必须登录后才能访问。

每个申请人档案都归当前登录账号单独管理。

### 3.1 基础信息

当前档案结构：

- 基础信息
  - 申请人姓名
- 美签信息内容
  - AA 码
  - 姓
  - 出生年份
  - 护照号
- 申根信息内容
  - 申根国家

### 3.2 材料分类

档案里的材料分成两类：

- 上传材料
  - 员工主动上传给系统使用的原始资料
- 递签材料
  - 自动化成功后生成并自动归档的成品材料

### 3.3 美签档案材料

上传材料：

- 美签照片
- DS-160 / AIS Excel

递签材料：

- DS-160 确认页 PDF

### 3.4 法签档案材料

上传材料：

- 申根照片
- 申根 Excel
- 护照扫描件

递签材料：

- 法国新申请 JSON
- 法国回执单 PDF
- 法国最终表 PDF

## 4. 美签操作 SOP

页面：

- `/usa-visa`

当前页签：

- 照片检测
- DS-160 填表
- 提交 DS-160
- AIS 注册

### 4.1 美签通用规则

- 一组 = 一个申请人
- 支持一次添加多组
- 每组都可以：
  - 直接使用当前申请人档案
  - 改选别的申请人档案
  - 或手动上传文件覆盖

### 4.2 推荐操作顺序

1. 进入 `/applicants`
2. 创建申请人档案
3. 上传：
   - 美签照片
   - DS-160 / AIS Excel
4. 进入 `/usa-visa`
5. 选中当前申请人
6. 先跑 `照片检测`
7. 再跑 `DS-160 填表`
8. 再跑 `提交 DS-160`
9. 最后跑 `AIS 注册`

### 4.3 美签自动更新规则

- 上传 DS-160 / AIS Excel 后，系统会自动识别并回写：
  - 姓
  - 出生年份
  - 护照号
- AA 码不能从 Excel 获取
- AA 码只能来自 DS-160 填表成功后的真实结果
- 提交 DS-160 成功后：
  - DS-160 确认页 PDF 会自动归档到申请人档案
- 照片检测成功后：
  - 如果系统生成了新的合格照片，会自动替换档案里的旧照片

### 4.4 美签注意事项

- 不要手动编造或录入 AA 码
- AA 码必须以 DS-160 填表成功结果为准

## 5. 法签操作 SOP

页面：

- `/schengen-visa/france/automation`

当前页签：

- 提取+注册
- 生成新申请
- 填写回执单
- 提交最终表

### 5.1 法签通用规则

- 法签页也已经改成申请组模式
- 一组 = 一个申请人
- 每组都可以：
  - 直接使用当前申请人档案
  - 改选别的申请人档案
  - 或手动上传 Excel 覆盖

### 5.2 法签注册规则

现在员工操作时，不再单独使用“账号注册”。

统一使用：

- `提取+注册`

含义是：

- 系统先提取 FV 注册信息
- 再继续执行 France-visas 注册流程

### 5.3 推荐操作顺序

1. 进入 `/applicants`
2. 创建申请人档案
3. 上传：
   - 申根 Excel
   - 申根照片（如需要）
   - 护照扫描件（如需要）
4. 进入 `/schengen-visa/france/automation`
5. 选中当前申请人
6. 跑 `提取+注册`
7. 跑 `生成新申请`
8. 跑 `填写回执单`
9. 跑 `提交最终表`

### 5.4 法签材料归档规则

自动化成功后：

- `生成新申请`
  - 现在只生成 JSON
  - 不再生成 Excel
  - JSON 会自动归档到申请人档案
- `填写回执单`
  - 生成的 PDF 会自动归档到申请人档案
- `提交最终表`
  - 生成的 PDF 会自动归档到申请人档案

## 6. 任务查看规则

美签页和法签页的每个操作区块下面都有任务列表。

推荐动作：

1. 提交任务
2. 在下方任务列表等待完成
3. 需要时从任务卡片下载结果
4. 再回 `/applicants` 检查最终材料是否已经归档

当前系统也支持按申请人过滤任务，新的任务会尽量正确挂到当前申请人名下。

## 7. 员工日常操作模板

一个新客户建议按这套流程走：

1. 登录系统
2. 打开 `/applicants`
3. 创建申请人档案
4. 上传初始资料
5. 选中当前申请人
6. 进入对应业务页执行自动化
7. 观察任务状态直到完成
8. 如有需要下载任务结果
9. 回档案中心确认递签材料是否已归档

## 8. 禁止事项

- 不要把多个申请人资料混进同一个档案
- 不要跳过申请人档案直接做自动化
- 不要手工乱填 AA 码
- 不要把自动生成的递签材料当成普通上传材料反复覆盖
- 不要以为服务器会弹出可见浏览器窗口，现在默认都是无头运行

## 11. 出问题时怎么反馈

先自查：

- 当前是否选中了正确的申请人档案
- 对应 Excel / 照片是否已经上传
- 任务是否真的创建成功
- 任务卡片里是否有报错信息
- 申请人档案里是否收到了最终材料

如果还解决不了，反馈给维护人员时请至少提供：

- 页面路径
- 申请人姓名
- 操作的是哪个页签
- 任务类型
- 报错信息
- 截图
